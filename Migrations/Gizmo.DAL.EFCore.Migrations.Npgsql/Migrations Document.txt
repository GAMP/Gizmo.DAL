- To add new migration:

	1- Configure the dbcontext as the following :

	   connectionString = "Host={Host};Database={Database};Username={Username};Password={Password}";

       builder.Services.AddDbContext<ApplicationDbContext>(options => 
	   options.UseNpgsql(connectionString, x => x.MigrationsAssembly("Gizmo.DAL.EFCore.Migrations.Npgsql"));
	   
	2- Open package manager console.
	3- Select the default project Gizmo.DAL.EFCore.Migrations.Npgsql
	4- Enter the command line > EntityFrameworkCore\Add-Migration {MigrationName}


- Remarks
	
	1- Postgre SQL requires the index name to be unique on database level not on table level so we are using this funtion "RenameDuplicatedIndexNames" in 
	   DefaultDbContext to fix this.
		
	2- Add the following before using Postgre SQL to overcome the UTC time issues with the new versions
		AppContext.SetSwitch("Npgsql.EnableLegacyTimestampBehavior", true);
